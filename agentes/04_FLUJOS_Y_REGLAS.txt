KC MACHINE SELLS CONTROL - FLUJOS Y REGLAS
Actualizado: 2026-06-03

FLUJO DE ARRANQUE

1. script.js define STORAGE_KEY.
2. loadState() lee localStorage.
3. Se normaliza el estado, maquinas, globalSettings, machineProductIds y machineId en registros viejos.
4. Se sincroniza catalogo con syncProductCatalog().
5. loadDate(visitDate.value) pinta la pantalla para la maquina activa, o estado cero si no hay maquina activa.
6. loadStateFromConvex() intenta traer estado remoto.
7. Si remoto gana, applyRemoteState() reemplaza estado local y repinta.

ESTADO CERO

Si activeMachineId es null o no existe maquina activa:
- No se puede guardar visita.
- La pantalla principal muestra un panel para crear maquina.
- El boton principal de la hamburguesa muestra "Sin maquina".
- La vista Configuracion permite crear la primera maquina.

No se crea kc-01 automaticamente.

SELECTOR DE MAQUINA

El boton machineSwitcher en la hamburguesa:
- Muestra el nombre de la maquina activa.
- Abre una lista de maquinas.
- Permite cambiar activeMachineId.
- Incluye acceso a crear maquina desde Configuracion.

Al cambiar de maquina:
- saveState() persiste activeMachineId.
- loadDate() repinta matriz, historial, resumen y cierres filtrados por machineId.

CREACION DE MAQUINA

Crear desde cero:
- Requiere nombre y una matriz visual valida.
- El diseñador permite hasta 12 filas y hasta 12 espacios por fila.
- Crea rowSizes propio de la maquina desde el diseñador.
- Crea visualLayout vacio: todos los espacios inician EMPTY.
- No copia settings, visitas, cierres, ventas ni stock de otra maquina.

Heredar de otra maquina:
- Requiere nombre y una maquina origen.
- Copia rowSizes de la maquina origen.
- Copia visualLayout desde la ultima visita de la maquina origen, o desde el layout guardado si aun no hay visita.
- No copia settings, visitas, cierres, ventas, stock ni datos contables.

Productos:
- El catalogo de productos es global y compartido por todas las maquinas.
- machineProductIds queda solo como compatibilidad legacy; la disponibilidad real sale de products.
- Que un producto este o no en una maquina se define por su posicion visual en la matriz.

REGLA LOCAL VS REMOTO

Si local tiene datos pero remoto esta limpio:
- Gana el que tenga updatedAt mas nuevo.

Esto protege los resets de Convex:
- Si se limpia Convex y el reset tiene updatedAt mas nuevo, local viejo no vuelve a subir.

FLUJO DE CARGA DE FECHA

Funcion:
- loadDate(date)

Hace:
1. Aplica cambios pendientes de catalogo si corresponden.
2. Precarga imagenes de productos.
3. Lee record existente de la maquina activa si hay.
4. Calcula stock anterior por producto usando registros/cierres de la maquina activa.
5. Construye currentSlots desde buildDefaultSlots(), usando rowSizes y visualLayout de la maquina activa.
6. Crea currentVisitDetails con createVisitDetailsForDate().
7. Aplica detalles a slots.
8. Calcula representantes.
9. Renderiza maquina y resumen.

CARGA INICIAL

Condicion:
- No hay record previo para la maquina activa.
- No hay stock anterior para la maquina activa.

Resultado al guardar:
- type = carga_inicial
- salesMovements = []
- visitDetails guarda NS.
- nextStock sale de NS.

VISITA REAL

Condicion:
- Existe stock anterior desde visita previa o cierre.

Regla:
- SM es obligatorio para cerrar si hay representante.
- NS es obligatorio para cerrar si hay representante.
- Si no hay representante y stockAnterior > 0, se solicita SM manual.
- Si no hay representante, NS queda 0.

CALCULO DE VENTA

UV = stockAnterior - SM

Si UV < 0:
- estado = error
- se bloquea guardado.

PRODUCTO SIN REPRESENTANTE

Caso A:
- stockAnterior = 0
- representative = SR

Resultado:
- SM = 0
- NS = 0
- UV = 0
- No requiere accion.

Caso B:
- stockAnterior > 0
- representative = SR
- captured = false

Resultado:
- validateRequiredCaptures() pide SM manual con prompt.
- NS queda 0.
- UV se calcula con el SM ingresado.

PRODUCTO CON REPRESENTANTE

Solo el representante abre dialogo.
El dialogo edita:
- SM
- NS

El producto queda capturado si:
- foundCaptured = true
- stockEntry = true

DUPLICADOS VISUALES

Los duplicados no capturan.
Mantienen selector para poder cambiar producto.
La app recalcula representantes despues de cambios visuales.

FUSION POR FILA

Funcion:
- applyVisualFusionByRow()

Si un producto se repite dentro de una fila:
1. Se agrupa desde el primer ejemplar.
2. Otros productos conservan orden relativo posible.
3. Solo cambia la vista.

FECHAS Y MESES BLOQUEADOS

Funcion:
- isDateLocked(date)

Bloquea si:
- El mes esta cerrado.
- La fecha ya tiene record guardado.

Motivo:
Evitar ediciones retroactivas mientras se estabiliza el modelo.

GUARDADO DE VISITA

Funcion:
- saveSelectedDate()

Antes de guardar:
0. Rechaza si no hay maquina activa.
1. Rechaza mes cerrado.
2. Rechaza fecha ya guardada.
3. Ejecuta validateRequiredCaptures().

Registro final:
- currentRecord()

currentRecord guarda:
- machineId
- date
- type
- status
- fuelCost
- catalogSnapshot
- slots
- slotsSnapshot
- visitDetails
- salesMovements
- nextStock

CIERRE MENSUAL

Funcion:
- finalizeMonthClosure()

Usa:
- accountingSummaryForRecords(records)
- recordNextStock(closingRecord)
- activeMachineSettings()

Guarda:
- monthClosures
- machineId
- openingStockNextMonth
- productTotals
- metricas de venta/costo/rentabilidad/gastos
- bankCommissionPercent usado
- floorFee usado como snapshot

La comision bancaria viene de machine.settings.bankCommissionPercent.
El derecho de piso se precarga desde machine.settings.monthlyFloorCost, pero el valor final guardado en el cierre puede editarse.
En el mes abierto, accountingSummaryForRecords aplica monthlyFloorCost solo si hay records en ese mes.
Los meses vacios del anio no suman derecho de piso en el panel anual.
Si el mes seleccionado ya esta cerrado, renderAccounting usa summaryFromClosure() para respetar el floorFee guardado como snapshot.

Si se cierra diciembre:
- crea yearClosures
- compacta meses del anio
- limpia records del anio cerrado solo de la maquina activa

CATALOGO

Cambios de catalogo global:
- Si la maquina activa no tiene historial ni cierres, se aplican inmediatamente para facilitar estado cero.
- Si ya hay historial/cierres, se programan para el siguiente mes mediante pendingProducts.

Motivo:
- Evitar cambiar columnas/precios dentro del mes actual.

Al agregar un producto desde Productos, se agrega al catalogo global compartido.
Para usarlo en una maquina, se selecciona directamente en la posicion visual de la matriz.
Los selectores visuales muestran activeProducts(), que actualmente devuelve todo products por ser catalogo global.

CONFIGURACION

Vista:
- settingsView

Global:
- Moneda.
- Espacio de idioma futuro.
- Espacio de preferencias visuales futuras.
- Crear maquinas desde cero o heredando productos.
- Inactivar/restaurar/eliminar maquinas.

Maquina actual:
- Nombre.
- Comision bancaria.
- Recordatorio de gasolina cada N visitas.
- Derecho de piso mensual.
- Exportar/importar backup de esa maquina.
- Reset de esa maquina con confirmacion por nombre.

RESET DE MAQUINA

Funcion:
- resetActiveMachineData()

Limpia:
- records de la maquina activa.
- monthClosures de la maquina activa.
- yearClosures de la maquina activa.

Conserva:
- Maquina.
- Settings.
- machineProductIds.
- Catalogo global.
- Productos, precios y costos.

ELIMINAR MAQUINA

Funcion:
- deleteMachinePermanently(machineId)

Requiere escribir el nombre de la maquina.
Borra esa maquina, su relacion machineProductIds y sus registros/cierres.
Permite que queden cero maquinas.

INFORMES

Los informes usan helpers compatibles:
- recordSalesMovements(record)
- recordNextStock(record)
- productTotalsForRecords(records)
- accountingSummaryForRecords(records)

Si record tiene visitDetails, los movimientos salen de UV y stock sale de NS.
Si record es viejo, hay fallback.
Los gastos operativos anuales suman floorFee desde cierres guardados o meses abiertos con visitas; no desde meses vacios.
