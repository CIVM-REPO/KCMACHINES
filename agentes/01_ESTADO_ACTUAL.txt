MODELO ACTUAL

La app mantiene un modelo hibrido:

1. appState
   Documento completo de compatibilidad. El frontend actual sigue leyendo/escribiendo este estado para mantener fluidez.

2. Tablas normalizadas Convex
   Se sincronizan desde appState en cada save. Preparan la base para reportes, auditoria y operacion multi-maquina.

La fuente logica de contabilidad en la app es visitDetails.

MODELO MULTI-MAQUINA ACTUAL

appState contiene:
- machines: lista de maquinas con id, name, status, deletedAt y settings.
- activeMachineId: maquina operativa actual, o null si no hay maquina activa.
- globalSettings: preferencias globales; hoy moneda, idioma futuro y preferencias visuales futuras.
- machineProductIds: compatibilidad legacy; el catalogo operativo es global en products.
- records/monthClosures/yearClosures con machineId en registros nuevos.

machine.settings contiene:
- bankCommissionPercent
- fuelReminderEveryVisits
- monthlyFloorCost

La comision bancaria, recordatorio de gasolina y derecho de piso mensual pertenecen a la maquina activa.
La configuracion global no contiene datos operativos.

ESTADO CERO

Si no hay maquina activa, la pantalla principal muestra un panel para crear maquina.
No se crea kc-01 automaticamente como fallback permanente.
El flujo esperado desde cero es:
1. Crear maquina.
2. Crear productos en el catalogo.
3. Asignar productos en espacios.
4. Guardar primera carga historica.
