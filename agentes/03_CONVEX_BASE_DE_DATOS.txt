SINCRONIZACION NORMALIZADA

Funcion:
- syncNormalizedState(ctx, appState)

Ubicacion:
- convex/appState.js

Se ejecuta dentro de appState.save.

Hace:
1. Normaliza maquinas.
2. Normaliza productos.
3. Elimina de tablas normalizadas datos de maquinas que ya no existan en appState.
4. Crea/actualiza maquina_productos de la maquina activa segun machineProductIds.
5. Reescribe matriz_visual de la maquina activa desde el ultimo record de esa maquina, o matriz vacia.
6. Reescribe visitas de la maquina activa desde records filtrados por machineId.
7. Reescribe detalle_visita desde record.visitDetails.

Importante:
- Actualmente la normalizacion reescribe visitas/detalles/matriz para la maquina activa.
- Esto simplifica consistencia mientras frontend sigue guardando appState completo.
- Los records sin machineId se migran en frontend hacia activeMachineId al normalizar estado viejo.
- Si no hay maquina activa, syncNormalizedState guarda maquinas/productos existentes pero no escribe matriz/visitas.

ESTADO CERO EN CONVEX

El schema y syncNormalizedState soportan:
- maquinas = 0
- productos = 0
- maquina_productos = 0
- matriz_visual = 0
- visitas = 0
- detalle_visita = 0

No se debe volver a introducir fallback obligatorio kc-01 en Convex.

RESET PARA PRUEBAS

Script:
- agentes/migracion/reset-convex-pruebas.mjs

Comando:
- npm.cmd run convex:reset-pruebas

Limpia:
- records
- monthClosures
- yearClosures
- pendingProducts
- pendingProductsEffectiveMonth
- seedScenario

Conserva:
- products
- machines
- activeMachineId

Nota:
El reset operativo de maquina desde la UI es diferente del script de pruebas.
El reset UI limpia solo records/monthClosures/yearClosures de la maquina activa y conserva maquina, settings, productos asignados y catalogo.
