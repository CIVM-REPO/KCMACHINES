PRUEBA 1 - CARGA INICIAL

Objetivo:
Verificar que primer guardado no genera ventas.

Pasos:
1. Resetear Convex.
2. Abrir app.
3. Asignar productos a algunos espacios.
4. Abrir representante de cada producto.
5. Dejar SM = 0.
6. Dejar NS > 0.
7. Guardar.

Esperado:
- Se crea 1 visita.
- tipo = carga_inicial.
- detalle_visita guarda NS por producto.
- UV = 0.
- ventas mensuales = 0.

PRUEBA 2 - SEGUNDA VISITA REAL

Objetivo:
Verificar ventas por producto.

Pasos:
1. Elegir fecha posterior.
2. La app carga stockAnterior desde NS previo.
3. Para un producto con stockAnterior 10, capturar SM = 4, NS = 8.
4. Guardar.

Esperado:
- UV = 6.
- detalle_visita tiene stockAnterior 10, SM 4, NS 8, UV 6.
- salesMovements reporta 6 vendidas.
- nextStock tiene stock 8 para ese producto.

PRUEBA 3 - PRODUCTO SIN REPRESENTANTE CON STOCK ANTERIOR

Objetivo:
Verificar retiro visual sin perder venta.

Pasos:
1. En una visita previa dejar stock de un producto.
2. En la siguiente visita quitar ese producto de la matriz.
3. Guardar.

Esperado:
- La app pide SM manual.
- NS queda 0.
- UV = stockAnterior - SM.
- El producto no desaparece de contabilidad.

PRUEBA 4 - PRODUCTO DUPLICADO

Objetivo:

Pasos:

3. Intentar abrir captura en duplicados.

Esperado:
- Solo el representante abre dialogo.
- Duplicados conservan selector.
- detalle_visita crea una sola fila por producto.

   convex deploy apunta a prod por defecto. Para el entorno usado por la app, usar npx.cmd convex dev --once.

   Si una prueba no refleja reset, abrir nueva pestana o limpiar localStorage. El codigo ya protege reset remoto mas nuevo.

3. Dos dispositivos editando a la vez.

4. Frontend aun no lee tablas normalizadas directamente.
   Las tablas se sincronizan como fuente preparada, pero appState sigue siendo compatibilidad principal.

5. Produccion.
